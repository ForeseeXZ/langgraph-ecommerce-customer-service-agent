# 工单优先级与SLA规则

## 适用场景

本 SOP 定义了所有售后工单的优先级判定规则、SLA 响应时效和处理组分配规则。当 Agent 调用 `create_after_sales_ticket` 时，系统自动根据这些规则设置工单优先级和分配组别。

## 优先级判定规则

工单优先级由问题类型、订单金额、会员等级三因素交叉决定：

### 紧急优先级（SLA：12 小时内首次响应）

| 触发条件 | 说明 |
|---------|------|
| 订单金额 ≥ 5000 元 | 任意问题类型 |
| 问题类型为"质量问题" 且订单金额 ≥ 2000 元 | 高价值质量问题 |
| 问题类型为"物流破损" 且订单金额 ≥ 2000 元 | 高价值物流损坏 |

### 高优先级（SLA：24 小时内首次响应）

| 触发条件 | 说明 |
|---------|------|
| 问题类型为"质量问题"（常规金额） | 低于 2000 元 |
| 问题类型为"物流破损"（常规金额） | 低于 2000 元 |
| 问题类型为"漏液" | 涉及安全隐患 |
| 问题类型为"过敏反馈" | 涉及人身健康 |
| 问题类型为"少件漏发" | 物流异常 |
| 订单金额 ≥ 1000 元（非上述类型） | 中高价值订单 |

### 中优先级（SLA：48 小时内首次响应）

| 触发条件 | 说明 |
|---------|------|
| "七天无理由""尺码不合适""未发货退款""催发货""安装问题""配件缺失""包装破损" | 一般售后问题 |
| 金卡/黑卡会员的"保修咨询""发票问题""地址修改" | 会员特权升级 |

### 低优先级（SLA：48 小时内首次响应）

| 触发条件 | 说明 |
|---------|------|
| 普通会员的"保修咨询" | 信息咨询类 |
| 普通会员的"发票问题" | 行政服务类 |
| 普通会员的"地址修改" | 订单管理类 |

## 处理组分配规则

| 处理组 | 负责的问题类型 |
|--------|-------------|
| **VIP客服组** | 黑卡会员的任意售后问题 |
| **仓储物流组** | 物流破损、少件漏发、催发货、地址修改 |
| **技术支持组** | 质量问题、漏液、过敏反馈、安装问题、配件缺失 |
| **退款审核组** | 七天无理由、未发货退款、尺码不合适 |
| **售后服务组** | 保修咨询、发票问题以及其他未分类问题 |

## 会员等级对优先级的影响

| 会员等级 | 影响规则 |
|---------|---------|
| 黑卡会员 | 直接分配 **VIP客服组**，绕过问题类型分配规则 |
| 金卡会员 | "保修咨询""发票问题""地址修改"优先级从"低"升级为"中" |
| 普通会员 | 按默认规则分配 |

## 工单状态流转（演示系统）

```
已创建 ──→ 待补充凭证 ──→ 审核中
                              │
                    ┌─────────┼─────────┐
                    ↓         ↓         ↓
                待寄回   退款中   换货处理中
                    │         │         │
                    ↓         ↓         ↓
              仓库验收中  退款成功  已完成
                    │
                    ↓
              退款中 → 退款成功
```

## SLA 时效明细（演示系统规则）

| 工单状态 | 处理时限 | 超时后果 |
|---------|---------|---------|
| 首次响应 | 12h/24h/48h（按优先级） | 工单自动升级至上级审核 |
| 补充凭证审核 | 收到凭证后 24 小时内 | 超时可联系用户催促 |
| 退款审核 | 退货签收后 7 日内 | — |
| 换货处理 | 审核通过后 48 小时内发货 | — |
| 维修完成 | 签收后 7-30 天（视故障复杂度） | 超 30 天可要求换货 |
| 工单关闭 | 退款到账/换货签收后 7 天自动关闭 | — |

## 推荐工具调用

所有工单创建都通过同一个入口：`create_after_sales_ticket(order_id, issue_type, description, contact_phone)`。优先级和分配组由系统基于上述规则自动计算，客服 Agent 无需手动设置。

查询工单进度：`query_after_sales_ticket(ticket_id=xxx)` 或 `query_after_sales_ticket(order_id=xxx)`。

## 注意事项

- 上述优先级和 SLA 均为演示系统内部规则，不代表真实电商平台的服务承诺。
- 会员等级从 `query_order_status` 返回的 `customer.member_level` 字段获取。
- 工单状态变更有对应的系统事件记录在 `ticket_events` 表中，可通过 `query_after_sales_ticket` 查看完整流转记录。
- 演示系统不支持工单关闭后的重新打开操作。
- 同一订单已有进行中工单时，`evaluate_refund_eligibility` 会返回 `eligible: false`，需引导用户查询现有工单进度而非重复创建。
