# 平台售后政策总则

## 适用场景

本 SOP 为平台售后服务的通用入口规则，适用于所有售后问题类型的初步处理。当用户首次提出售后诉求但问题类型尚不明确时，应先参照本总则进行初步判断。

## 客服处理原则

1. **先查后判**：先调用 `query_order_status` 获取订单事实（状态、签收时间、物流、已有工单），再判断售后资格。
2. **结构化优先**：调用 `evaluate_refund_eligibility` 获取结构化预判断结果，不可跳过该步骤直接承诺。
3. **政策兜底**：结构化工具无法给出明确结论时，调用 `query_refund_policy` 或查询 RAG 政策文档补充判断。
4. **创建工单**：前置判断通过后调用 `create_after_sales_ticket` 创建工单，将审核权交给人工。
5. **不做承诺**：演示系统内禁止承诺退款金额、退款到账时间、换货发出时间等具体结果。
6. **高风险必升人工**：遇到超出结构化规则覆盖范围的场景，提示用户将转交人工处理。

## 适用条件

以下条件全部满足时，可进入售后流程：
- 订单状态为"已发货""运输中""已签收"或"已完成"（未发货退款单独处理）
- 订单无进行中的售后工单（状态为已创建/待补充凭证/审核中/待寄回/仓库验收中/退款中/换货处理中）
- 用户能提供订单号和手机号后四位进行身份核验
- 售后问题类型在 15 种支持范围内

## 不适用或需人工审核的情况

- 订单状态为"待支付"或"已取消" — 不可创建普通售后工单
- 该订单已有进行中的售后工单 — 引导查询现有工单进度
- 订单已退款成功 — 不可再次申请
- 超过 7 天且问题类型为"七天无理由"或"尺码不合适" — 人工审核
- 商品标记为不支持退款（`refund_eligible = 0`）且非质量问题 — 人工审核
- 食品/化妆品已拆封影响安全卫生 — 人工审核
- 用户无法提供订单号或手机号后四位 — 引导获取凭证

## 用户需提供的凭证

| 凭证类型 | 适用场景 | 说明 |
|---------|---------|------|
| 订单号 | 全部 | 必须提供，用于第一步查询 |
| 手机号后四位 | 全部 | 身份核验，证明是买家本人 |
| 商品照片 | 质量问题、尺码不合适 | 整体 + 细节至少 2 张 |
| 包装照片 | 物流破损、包装破损 | 含面单和破损处 |
| 故障视频 | 质量问题（数码/家电） | 展示故障现象的短视频 |
| 过敏部位照片 | 过敏反馈 | 需清晰可见过敏部位 |
| 发货清单照片 | 少件漏发、配件缺失 | 实物与清单的对比 |

## 处理流程

1. **身份核验**：向用户获取订单号和手机号后四位。
2. **查询订单**：调用 `query_order_status(order_id, phone_last4)` 获取订单全貌。
3. **资格判断**：调用 `evaluate_refund_eligibility(order_id, issue_type)` 获取结构化资格结果。
4. **政策查询**：如资格判断返回 `manual_review`，调用 `query_refund_policy(issue_type, category)` 或 RAG 政策工具补充判断。
5. **创建工单**：调用 `create_after_sales_ticket(order_id, issue_type, description, contact_phone)` 建立售后记录。
6. **告知用户**：回复工单编号、处理组别、预计响应时效、需补充的凭证清单。
7. **进度可查**：告知用户可通过工单号随时查询进度。

## 处理时效（演示系统规则）

| 优先级 | 首次响应时效 | 适用条件 |
|--------|------------|---------|
| 紧急 | 12 小时内 | 订单金额 ≥ 5000 元，或质量问题/物流破损且金额 ≥ 2000 元 |
| 高 | 24 小时内 | 质量问题、物流破损、漏液、过敏反馈、少件漏发，或订单金额 ≥ 1000 元 |
| 中 | 48 小时内 | 一般售后问题 |
| 低 | 48 小时内 | 发票问题、地址修改等非紧急问题 |

> 以上时效均为演示系统的内部 SLA 规则，不代表真实退款或发货时间。

## 推荐工具调用

| 步骤 | 工具函数 | 说明 |
|------|---------|------|
| 查订单 | `query_order_status` | 获取订单、物流、已有工单 |
| 判资格 | `evaluate_refund_eligibility` | 结构化资格预判断 |
| 查政策 | `query_refund_policy` 或 RAG 检索 | 补充政策规则 |
| 建工单 | `create_after_sales_ticket` | 创建售后工单 |
| 查进度 | `query_after_sales_ticket` | 查询已有工单状态 |

## 注意事项

- 本 SOP 为演示系统内部规则，不构成真实平台的售后承诺。
- 所有退款、换货、赔偿结果以人工审核为准，客服 Agent 仅负责信息收集和工单创建。
- 如果 `evaluate_refund_eligibility` 返回 `eligible: false`，必须如实告知用户原因，不可绕过结构化判断。
- 同一订单不允许创建多个并行售后工单，必须先处理已有工单。
